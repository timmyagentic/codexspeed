# GPT-5.6 Series Run Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Every behavior change starts with a failing test.

**Goal:** Add a first-class, reproducible `gpt-5.6` series benchmark scope, publish Runner v0.1.3 and a compatible website, then execute and upload the complete 48-turn benchmark from the immutable release.

**Architecture:** The runner resolves a bounded series prefix against the live App Server catalog and creates a standard schedule containing every visible non-Ultra cell in that series. The strict shared contract validates the complete matrix and carries the series identifier through Worker/D1 read models to a single UI scope-label formatter. Existing schema-version-1 smoke and full artifacts remain valid.

**Tech Stack:** Node.js 22, pnpm 10, TypeScript, Zod, Vitest, Codex App Server JSON-RPC, Hono, Cloudflare Workers + D1, React, Playwright, GitHub Releases.

## Global constraints

- The approved matrix is GPT-5.6-Sol, GPT-5.6-Terra, and GPT-5.6-Luna at `low`, `medium`, `high`, `xhigh`, and `max`: 15 cells.
- A standard series run has one warm-up per model plus three measured rounds per cell: 48 sequential turns.
- `ultra`, hidden models, unsupported efforts, retries, Fast service tier, and arbitrary prompts remain excluded.
- `--series` is mutually exclusive with `--model`, `--effort`, `--no-warmup`, and any measured-round count other than three.
- Every plan/run invocation retains the mandatory `--max-turns` guard; the production run uses `--max-turns 48 --seed 13`.
- The website remains display-only and never receives Codex/OpenAI credentials.
- No benchmark turn starts until the compatible site is merged, released, deployed, and a detached v0.1.3 build passes doctor plus no-turn plan verification.
- Publication must use the existing Keychain HMAC secret without printing it. The generated artifact must remain private to the local user and pass the deny-list audit.
